# Scenario Summary: iam-attachgrouppolicy

**Created:** 2025-10-16
**Type:** One-hop privilege escalation to admin
**Environment:** Production account only

## Overview

This scenario demonstrates a privilege escalation vulnerability where an IAM user has permission to attach managed policies to a group they are a member of. By attaching the `AdministratorAccess` managed policy to their own group, the user gains administrator privileges through group membership.

## Attack Path

```
IAM User (pl-prod-one-hop-agp-starting-user) → Member of Group
→ iam:AttachGroupPolicy → Attach AdministratorAccess to Group
→ User gains Admin Access via Group Membership
```

## Resources Created

1. **IAM User**: `pl-prod-one-hop-agp-starting-user` (starting user with access keys)
2. **IAM Group**: `pl-prod-one-hop-agp-group` (group that user is member of)
3. **IAM Policy**: Inline policy allowing `iam:AttachGroupPolicy` on the group

## MITRE ATT&CK Mapping

- **Tactic**: Privilege Escalation
- **Technique**: T1098.003 - Account Manipulation: Additional Cloud Roles
- **Description**: Modifying group policies to gain elevated privileges

## Detection

CSPM tools should detect:
- IAM user can attach policies to groups they are members of
- Privilege escalation path via group policy attachment
- Overly permissive `iam:AttachGroupPolicy` permissions
- CloudTrail `AttachGroupPolicy` API calls adding administrative policies
